    Browsers automatically add a white space (margin) before and after a paragraph. Spaces, lines and line breaks are ignored.

  4. <pre> element

    The <pre> element represents preformatted text. tab indents, double-spaces, newlines, and other typographic formatting are preserved.
    It can be used to show code snippets. 

              text is
          kept as typed (newline!)
          double  space!
            tab   indent!
    Example with code
              <nav class="main-navigation">
          <ul>
          <li><a href="/">Home</a></li>
          <li><a href="about.html">About Us</a></li>
          <li><a href="contact.html">Contact Us</a></li>
          </ul>
          </nav>

  5. <address> element

    The <address> element defines the contact information for the author/ownder of a document (<body>) or article(<article>).
    Browser usally render the address in italic with a line break before and after.

    You can contact author at www.somedomain.com.
    If you see any bugs, please contact webmaster.
    You may also want to visit us:
    Mozilla Foundation
